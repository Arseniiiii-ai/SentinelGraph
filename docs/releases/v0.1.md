# SentinelGraph v0.1 — Data Acquisition and Temporal Splitting

## Release outcome

v0.1 establishes the reproducible and leakage-aware data foundation for later
models. The full PaySim1 version 2 distribution was acquired from Kaggle,
verified, profiled, validated, and split into typed Parquet artifacts.

## Delivered

- Version-pinned public Kaggle acquisition with atomic download.
- Source, CC BY-SA 4.0 dataset license, GPL-3.0 simulator-code distinction,
  citation, byte sizes, and SHA-256 metadata.
- Strict explicit CSV types and a release-blocking raw data contract.
- Exact full-data missingness, duplicate, label, temporal, and entity queries.
- Quantified semantic missingness for merchant destination balances.
- Point-in-time data dictionary and modelling leakage policy.
- Chronological train and future-time holdout.
- Future unseen-origin account evaluation slice.
- Checksummed Zstandard Parquet split artifacts.
- Generated Markdown data-quality report and machine-readable JSON results.
- Unit and integration coverage with Ruff and strict mypy validation.

## Key observations

- Rows: 6,362,620
- Steps: 1–743
- Fraud: 8,213 rows (0.1291%)
- Missing cells: 0
- Exact duplicate rows: 0
- Zero-amount rows: 16, all fraud-labelled `CASH_OUT`
- Train: steps 1–520, 6,082,007 rows
- Future: steps 521–743, 280,613 rows
- Future unseen-origin slice: 279,847 rows

The very high unseen-origin share and the increase in label prevalence in the
future window are dataset characteristics that later releases must report, not
smooth away with random splitting.

## Release boundary

No predictive model is trained in v0.1. Balance fields and policy outputs are
explicitly excluded from the upcoming learned baselines because the dataset
publisher warns that fraud cancellation affects the balance state.
