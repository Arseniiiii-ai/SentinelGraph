# SentinelGraph v0.2 — Leakage-Safe Baseline Models

## Release outcome

v0.2 stops at reproducible baseline models. It adds no behavioural, anomaly,
graph, API, calibration, or decision-engine functionality.

## Delivered models

- Label-prior `DummyClassifier` sanity check.
- Transparent policy rule recomputed from transaction type and amount.
- Balanced and scaled logistic regression.
- Balanced histogram gradient boosting.

## Evaluation design

- Development fitting: steps 1–416.
- Temporal threshold validation: steps 417–520.
- Future-time evaluation: steps 521–743.
- Cold-start view: future-time rows with unseen origin accounts.
- Learned thresholds target no more than 1% validation FPR.
- The rule keeps its fixed policy threshold so its operational cost remains
  visible.

## Key future-time results

| Model | PR-AUC | Recall | FP / 10k legitimate | Captured fraud amount |
| --- | ---: | ---: | ---: | ---: |
| Large-transfer rule | 0.02019 | 33.47% | 649.97 | 49.72% |
| Logistic regression | 0.21612 | 29.56% | 58.24 | 74.45% |
| Histogram gradient boosting | 0.40481 | 47.45% | 56.08 | 80.11% |

Gradient boosting is the strongest v0.2 baseline, but the probabilities are not
calibrated and PaySim is synthetic. The results establish an engineering
comparison point, not production fraud performance.

## Reproduce

```bash
uv sync --extra dev
uv run sentinelgraph-data all
uv run sentinelgraph-baselines
```

Tracked metrics are stored in `reports/v0.2/baseline_metrics.json`; trained
model binaries remain ignored under `models/v0.2/`.
